Data Analysis Insight - Demographics

👤 กลุ่มผู้ใช้หลัก:
อายุเฉลี่ยอยู่ในช่วง 30-46 ปี ซึ่งเป็นกลุ่ม Millennial ถึง Gen X — นี่คือกลุ่มที่มีความพร้อมทางการเงินและมักจะซื้อบ้านเพื่ออยู่อาศัยจริงหรือลงทุน ไม่ใช่แค่ดูเล่น
⚤ สัดส่วนเพศ:
ผู้ใช้ส่วนใหญ่เป็นเพศชาย — ชี้ให้เห็นว่า UX/UI ปัจจุบัน (การค้นหาแบบกรอกข้อมูลละเอียด) อาจเหมาะกับสไตล์การตัดสินใจของเพศชายมากกว่า ควรพิจารณาทำ Social Proof หรือข้อมูลภาพถ่ายให้ดึงดูดเพศหญิงด้วย
🔍 ข้อสังเกต:
พบ record บางส่วนมีอายุน้อยผิดปกติ (1-14 ปี) — น่าจะเป็นข้อมูลทดสอบ (Test Data) ในช่วง Dev ต้องกรองออกก่อนวิเคราะห์ เพื่อความแม่นยำ

Business Insight - Budget & Demand

💰 งบประมาณที่น่าจะตรงกลุ่ม:
กลุ่มอายุ 30-46 ปีในไทย โดยเฉลี่ยมีความสามารถกู้สินเชื่อ ฿3-8 ล้านบาท สำหรับที่อยู่อาศัย แต่จากข้อมูล Supply ของเรา ทรัพย์ส่วนใหญ่ที่มีในระบบมักเป็นระดับ Premium (>8M) ทำให้เกิด Mismatch
📍 โซนที่น่าสนใจ (28 โซน):
ระบบรองรับทำเลหลากหลายทั้ง CBD (Silom, Asok, Nana) และชานเมือง (Bang Bon, Ram Inthra) แต่ยังไม่มีข้อมูล Heatmap ว่าโซนไหน "Search มากที่สุด" ต้องเปิด GA4 Audience Explorer เพื่อยืนยัน
⚠️ ปัญหาเชิงธุรกิจ:
Budget_min และ Budget_max ใน API เป็น 0 ทั้งหมด — แปลว่าระบบยังไม่ได้เก็บงบประมาณของ user จาก Flow ปัจจุบัน ต้องเพิ่ม field นี้ใน /analyze endpoint เพื่อให้วิเคราะห์ได้ละเอียดยิ่งขึ้น

💡 3 สาเหตุหลักที่ยังไม่มีคนจอง

วิเคราะห์จากข้อมูลจริง: GA4 CSV + MySQL DB (api_logs + bookings + property_specs)

🔴

1. ระบบพัง — Tech Errors

Web Exceptions พุ่งสูงถึง 6,836 ครั้ง จากผู้ใช้ 328 คน = เฉลี่ย 20.8 ครั้ง/คน แปลว่าระบบพังแทบทุกครั้งที่ลูกค้าพยายามโต้ตอบ ทำให้เกิด Distrust และ Churn ทันที

🟡

2. หน้าแรกดึงดูดไม่พอ

จาก Home 1,762 คน ไป Search แค่ 226 คน (12.8%) ชี้ว่าหน้าแรกขาด Clear CTA หรือ Value Proposition ที่โน้มน้าวพอ — คนส่วนใหญ่เข้ามาแล้วออกทันทีโดยไม่ลองใช้

🟣

3. Supply ไม่ตรงความต้องการ

166 คน รันผล AI และอ่านนาน ~8 นาที (503 วินาที) แต่มีแค่ 33 คน (19.9%) กดจอง — สัญญาณชัดว่า "บ้านที่ได้ไม่ตรงงบหรือทำเล" ที่ต้องการจริงๆ
